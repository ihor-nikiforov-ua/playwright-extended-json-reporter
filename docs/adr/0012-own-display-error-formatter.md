# Own Display Error Formatter

The Runboard Reporter treats `result.errors[]` as the Playwright-compatible Display Error surface, while `result.runboard.evidence[]` remains index-aligned structured enrichment for Runboard-specific classification, linking, grouping, and history. Release-grade quality requires Display Error parity across every Error Type in the Error Catalog with only minimal non-semantic normalization, but production code will use a Runboard Reporter-owned formatter built from public Playwright reporter API data rather than runtime private Playwright formatter imports; Playwright's official HTML reporter remains the oracle in Compatibility Fixtures. Optional Source Excerpts may be added to Structured Error Evidence as a schema `1.1.0` expansion so Runboard can render custom codeframes without parsing Display Errors, and `noSnippets: true` suppresses those source-bearing excerpts.
